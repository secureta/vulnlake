# Lint / SAST 導入 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ruff (lint + format + S ルール) / actionlint / zizmor を導入し、CI の lint ジョブで自動チェックする。

**Architecture:** Python は Ruff 一本(lint + format + flake8-bandit 相当の S ルール)。GitHub Actions は actionlint (lint) + zizmor (SAST)。CodeQL default setup は有効化済みで、これらが補完する。CI は既存 `test.yml` に `lint` ジョブを追加し `test` ジョブと並列実行。

**Tech Stack:** Python 3.12+ / uv / ruff 0.15.21 / zizmor 1.26.1 / actionlint 1.7.12

**Spec:** `docs/superpowers/specs/2026-07-11-lint-sast-design.md`

## Global Constraints

- 依存追加はレジストリ確認済みの最新安定版: `ruff>=0.15.21`, `zizmor>=1.26.1`(いずれも dev group、renovate が uv.lock 経由で追従)
- actionlint は `1.7.12` を CI で download script 取得(バージョン引数で固定)
- 全ワークフローで `permissions: contents: read` を維持
- action の `uses:` はコミットハッシュでピン留めし `# vX` コメントを付ける(publish.yml の既存流儀)
- コメントは既存コードに合わせて日本語

            # META はクラス定数の固定識別子で外部入力は入らない

        ).fetchall()

### Task 5: CI に lint ジョブを追加

**Files:**
- Modify: `.github/workflows/test.yml`

**Interfaces:**
- Consumes: Task 1-4 で green になった `uv run ruff check .` / `uv run ruff format --check .` / `uv run zizmor .github/workflows/`

- [ ] **Step 1: test.yml に lint ジョブを追加**

`jobs:` 配下、`test` ジョブと並列に:

```yaml
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@9c091bb21b7c1c1d1991bb908d89e4e9dddfe3e0 # v7
        with:
          persist-credentials: false
      - uses: astral-sh/setup-uv@11f9893b081a58869d3b5fccaea48c9e9e46f990 # v8.3.2
      - run: uv sync
      - run: uv run ruff check .
      - run: uv run ruff format --check .
      - run: uv run zizmor --no-progress .github/workflows/
      - name: actionlint
        run: |
          bash <(curl -sSfL https://raw.githubusercontent.com/rhysd/actionlint/v1.7.12/scripts/download-actionlint.bash) 1.7.12 "$RUNNER_TEMP"
          "$RUNNER_TEMP/actionlint" -color
```

- [ ] **Step 2: 変更後のワークフローを再検証**

Run: `uv run zizmor --no-progress .github/workflows/`
Expected: 0 findings

actionlint がローカルで実行できる場合(バイナリ取得の許可が出た場合)は `.github/workflows/` に対して実行し 0 findings を確認。できない場合は CI での初回実行で確認する(その旨を最終報告に明記)。
