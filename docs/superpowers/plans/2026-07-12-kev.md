# KEV データセット追加 実装計画

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** CISA KEV カタログ (JSON) を、内容比較差分の update のみ(backfill 無し)で vlake に取り込む。

**Architecture:** 既存のデータセットプラグイン構造に倣い `kev.py` を追加。レコード単位の更新日時が無いため、カタログ latest とのフィールド直接比較で差分追記する。削除はトゥームストーン行 (`removed=true`)、初回 update が全量投入となる。方式は nuclei (`docs/superpowers/specs/2026-07-12-nuclei-design.md`) を踏襲し、digest 計算だけ不要 (平坦な固定フィールドの dict 比較で足りる)。

**Tech Stack:** Python >= 3.14, uv, pyarrow, duckdb (DuckLake), httpx, click, pytest, ruff, zizmor(依存追加なし)

**Spec:** `docs/superpowers/specs/2026-07-12-kev-design.md`

## Global Constraints

- コード内のコメント・docstring・コミットメッセージは日本語。
- **公開順序の不変条件**: Parquet データファイルを先にアップロードし、カタログ (`vlake.ducklake`) の差し替えは必ず最後。
- 名称: データセット名・CLI・テーブル接頭辞は `kev`。`vlake` の命名は変更しない。
- 各タスク完了時に `uv run pytest -v` 全緑、`uv run ruff check .`・`uv run ruff format --check .` クリーン。workflows を触るタスクでは `uv run zizmor .github/workflows/` も実行。
- kev に **backfill は無い**(初回 update が全量投入)。`backfill.yml` / backfill CLI には追加しない。
- テストはネットワークに出ない(`kev.download` を monkeypatch)。
